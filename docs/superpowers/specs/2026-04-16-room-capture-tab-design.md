# 分间采集 Tab 设计说明

日期：2026-04-16

## 背景与目标

现有 Demo 具备多种相机功能（参数相机、mesh 相机、双目相机等），本次新增“分间采集”功能，用于引导一线资管人员按 SOP 进行房源视频采集。目标是让非专业用户在强引导与实时纠错下稳定产出合格素材。

## 产品范围

### 目标

- 新增第 5 个 Tab「分间采集」，不影响现有功能。
- 进入即为采集流程页（无中间首页）。
- 首版支持四项引导与提示：速度档位、角度提示、步骤进度、圈数/遍数。
- 步骤推进采用“系统估计完成 + 用户确认下一步”。
- 房间清单来自后端；首版用测试数据占位。

### 非目标

- 不引入 ARKit/LiDAR。
- 不做实时图像质检与模糊检测。
- 不做复核页的完整逻辑（首版仅入口/按钮）。
- 不实现真实 m/s 速度测量。

## 用户流程

1. 用户点击 Tab「分间采集」进入采集流程页。
2. 顶部显示房间名称与当前步骤。
3. 用户点击录制开始，系统展示实时速度档位、角度提示、圈数/遍数、步骤进度。
4. 系统在满足最低时长与动作阈值后提示“可进入下一步”。
5. 用户点击“下一步”确认进入下一步骤。
6. 完成步骤 1-4 后进入步骤 5（复核入口），用户点击“完成房间”进入下一个房间。

## 采集步骤定义

- 步骤 1：入门稳定（静止 2-3 秒）
- 步骤 2：边缘绕行（绕行 2 圈，视角上下各一遍）
- 步骤 3：中心小环（原地旋转 2 遍，视角上下各一遍）
- 步骤 4：穿行扫描（小/中/大房间按路径模板）
- 步骤 5：复核确认（首版仅按钮与提示）

## UI 布局

- 顶部条：房间名 + 步骤名；右侧显示参数锁定标签（0.5x / 1080P / 30fps）。
- 取景区：占画面 70-75%，叠加低对比构图框与参考线。
- 左侧浮层：速度档位（慢/中/快 + 绿黄红），角度提示（目标 5-10 度下俯）。
- 右侧浮层：圈数/遍数进度，步骤总进度（步骤 X/5）。
- 底部栏：录制按钮、步骤提示文案、下一步按钮（默认隐藏，满足条件显示）。

## 可落地技术策略（iOS）

### 角度提示

- 使用 CoreMotion 获取 pitch/roll。
- 目标区间：下俯 5-10 度；超限时提示并支持轻震动。

### 速度提示（相对档位）

- 不计算真实速度。
- 使用 IMU 运动强度指标（加速度 RMS + 角速度 RMS）。
- 映射为三档：慢/中/快，并给出绿/黄/红提示。

### 圈数/遍数估计

- 绕行/小环：使用 yaw 累积角度估计 360 度为 1 圈，支持误差容忍。
- 穿行扫描：用时间 + 方向变化次数估计遍数。

### 步骤完成判定

- 以“最低时长 + 基本动作阈值”作为完成条件。
- 满足条件后提示“可进入下一步”，用户点击确认推进。

## 性能与兼容性

- 只做 IMU 轻量计算，不做实时图像分析。
- 保证引导层叠加后帧率不低于 28fps。
- iOS 13+ 兼容。

## 错误与容错

- IMU 采样异常时，提示“传感器数据异常，请稍候重试”。
- 用户中断录制：保留当前步骤状态，可重新开始该步骤。

## 测试要点

- IMU 数据异常与权限拒绝的提示是否出现。
- 速度档位切换阈值是否稳定，不频繁抖动。
- 角度提示在 5-10 度范围内是否正确显示。
- 步骤推进按钮的出现/隐藏是否符合阈值逻辑。
- UI 叠加是否遮挡取景主体。

## 结论

本方案在不引入 ARKit/LiDAR 的前提下可落地，核心指标通过 IMU 相对估计实现。首版采用“系统估计 + 用户确认”的推进方式，降低误判风险，同时保持操作可控与引导性。
